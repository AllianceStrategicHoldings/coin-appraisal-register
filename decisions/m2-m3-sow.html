How this number was reached. The revised spec shifted in both directions versus the May 12 proposal. Removed: bidirectional GHL ID capture, barcode-scanner SDK, full compliance automation (LE reporting, structuring, Form 8300, 1099-B), Suspicious Item flag, Estate Lot bulk-estimate mode, 24-hour spot-change indicator, picker coin images and recently-used row. Added: Dual Pricing Engine (Max Payout / Actual Offer / Negotiation Delta), cloud storage in S3 or R2, webhook fan-out to four destinations with independent retry, Multi-Location and Roadshow mode, Override Code Mechanism, Manager Dashboard with per-rep performance, Test Mode, Customer_Master and returning-customer lookup, offline degradation, and a five-state deal status workflow. Net: scope is meaningfully larger than the prior proposal, and the SOW reflects that.

Section 5 — Boundary of Engagement (acknowledged)

The hand-off boundaries below are confirmed in full. Items in this list are not in scope under this SOW; they are either handled by other parties, are operator costs, or are explicitly deferred:

  • Inside-GHL configuration (workflows, SMS templates, follow-up sequences, opportunity pipeline configuration)
  • CRM-side configuration (mapping, dashboards, reports in the operator's primary CRM)
  • Hardware procurement (iPads, stands, scales)
  • Branding finalization (logo, accent color, app name — placeholders during build; final assets land via follow-up update)
  • Recurring third-party costs (PCGS, CDN Greysheet, Metals-API, Apify, Make.com Pro, Vercel, primary database, cloud storage)
  • Compliance automation (state law-enforcement reporting, structuring detection, Form 8300 automation, 1099-B threshold tracking and filing, encrypted PII vault)
  • Native iOS push notifications (5-second polling per Section 3.3 is the manager-notification mechanism)
  • Post-launch new features (the M3 PWA milestone includes 3 months of bug-fix support; new features are separate scope)
